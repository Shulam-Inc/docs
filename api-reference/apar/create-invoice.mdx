---
title: 'Create Invoice'
api: 'POST /api/v1/apar/invoices'
description: 'Create a new AP (payable) or AR (receivable) invoice'
---

## Request Body

<ParamField body="businessId" type="string" required>
  Your business identifier
</ParamField>

<ParamField body="direction" type="string" required>
  `payable` (you owe vendor) or `receivable` (customer owes you)
</ParamField>

<ParamField body="issuerAddress" type="string" required>
  Ethereum address of the invoice issuer (seller/vendor). Lowercase.
</ParamField>

<ParamField body="recipientAddress" type="string" required>
  Ethereum address of the recipient (buyer/customer). Lowercase.
</ParamField>

<ParamField body="amountAtomic" type="string" required>
  Amount in atomic USDC units (1 USDC = 1,000,000). Example: `"500000000"` = $500.
</ParamField>

<ParamField body="terms" type="string" default="net_30">
  Payment terms: `due_on_receipt`, `net_7`, `net_15`, `net_30`, `net_60`, `net_90`
</ParamField>

<ParamField body="lineItems" type="array">
  Optional line items with description, quantity, unitPriceAtomic, totalAtomic
</ParamField>

<ParamField body="dueDate" type="string">
  ISO-8601 due date. Auto-calculated from terms if omitted.
</ParamField>

<ParamField body="metadata" type="object">
  Optional key-value metadata (PO number, project code, etc.)
</ParamField>

## Response

```json
{
  "invoice": {
    "invoiceId": "a1b2c3d4-...",
    "invoiceNumber": "INV-2026-001",
    "businessId": "biz-001",
    "direction": "payable",
    "status": "approved",
    "issuerAddress": "0xvendor...",
    "recipientAddress": "0xbuyer...",
    "amountAtomic": "500000000",
    "dueDate": "2026-04-15T00:00:00.000Z",
    "terms": "net_30",
    "approvalChain": [],
    "createdAt": "2026-03-15T12:00:00.000Z"
  }
}
```

**Auto-approval:** Invoices below $1,000 USDC are auto-approved. Above threshold → `pending_approval` status.
